# HLD — UX Polish and Navigation

## Overview

Four focused features that improve how users share views, save favourite places, navigate the app with a keyboard, and read coordinates under the cursor. All are client-only, use existing browser APIs and MapLibre, add no new dependencies, and follow the existing module-per-feature pattern with minimal wiring in `main.ts`.

## Features

### 1. Deep-Link URL Sharing

#### What it does

Encodes the current camera state and active toggles into the URL hash so that copying the URL captures the exact view. Loading a URL with a hash restores that view, overriding default camera position and toggle state. Only non-default values are written, keeping URLs short.

#### Design

**New file:** `src/cameraHash.ts`

**Types and constants:**

```ts
type CameraHashState = {
  lat?: number;
  lng?: number;
  z?: number;    // zoom
  p?: number;    // pitch
  b?: number;    // bearing
  terrain?: boolean;
  night?: boolean;
  weather?: boolean;
  relief?: boolean;
  buildings?: boolean;
  spin?: boolean;
};

const DEFAULTS: Required<CameraHashState>;  // mirrors initial mapState + camera from main.ts
const HASH_DEBOUNCE_MS = 400;
```

**Key functions:**

```ts
/** Parse window.location.hash into a partial CameraHashState. Returns empty object if hash is absent or malformed. */
export function parseHash(hash: string): CameraHashState;

/** Serialize a CameraHashState to a URL hash string, omitting keys that match DEFAULTS. Returns "" if all defaults. */
export function serializeHash(state: CameraHashState): string;

/** Round a float to fixed decimals for URL brevity. */
export function roundForHash(value: number, decimals: number): number;
```

Hash format: `#lat=55.9533&lng=-3.1883&z=14.8&p=68&b=-20&terrain=1&night=1`

Boolean toggles are encoded as `1` (on) or `0` (off), omitted when matching the default.

**Integration with `main.ts`:**

- In `bootstrap()`, after map creation but before `wireMap()`: call `parseHash(window.location.hash)` and if non-empty, apply camera values to the `flyTo` / initial center/zoom/pitch/bearing, and apply toggle booleans to `mapState` + sync toggle chip classes.
- In `wireMap()`, on the existing `moveend` handler: call a debounced function that reads current camera + `mapState` toggles, serializes, and writes `window.location.hash` via `history.replaceState()` (avoids polluting browser history).
- No `hashchange` listener needed — hash is write-only after initial parse.

**CSS changes:** None.

#### Test plan

Tests in `src/cameraHash.test.ts`. All pure functions, no DOM needed.

- `parseHash("")` returns empty object.
- `parseHash("#lat=55.95&lng=-3.19&z=14.8")` returns `{ lat: 55.95, lng: -3.19, z: 14.8 }`.
- `parseHash("#terrain=0&night=1")` returns `{ terrain: false, night: true }`.
- `parseHash` ignores unknown keys and malformed values (NaN, empty strings).
- `serializeHash` omits keys matching defaults.
- `serializeHash` with all defaults returns `""`.
- `roundForHash` rounds lat/lng to 4 decimals, zoom to 1, pitch/bearing to 0.
- Round-trip: `parseHash(serializeHash(state))` reproduces the state within rounding tolerance.

### 2. Bookmarks / My Places

#### What it does

Lets users save the current camera position with a custom name to `localStorage`. Saved bookmarks appear in the preset grid alongside the built-in Launch Views. Clicking a bookmark flies to the saved position. Users can delete bookmarks they no longer need.

#### Design

**New file:** `src/bookmarks.ts`

**Types and constants:**

```ts
/** Reuses the shape of Preset from main.ts, adding a user flag. */
type Bookmark = {
  id: string;         // crypto.randomUUID()
  label: string;
  caption: string;    // auto-generated from coordinates, e.g. "55.95, -3.19"
  lng: number;
  lat: number;
  zoom: number;
  pitch: number;
  bearing: number;
};

const STORAGE_KEY = "worldviewer-bookmarks";
const MAX_BOOKMARKS = 24;
```

**Key functions:**

```ts
/** Load bookmarks from localStorage. Returns [] on parse failure. */
export function loadBookmarks(): Bookmark[];

/** Save bookmarks array to localStorage. */
export function saveBookmarks(bookmarks: Bookmark[]): void;

/** Create a Bookmark from the current camera state and a user-supplied name. */
export function createBookmark(
  label: string,
  camera: { lng: number; lat: number; zoom: number; pitch: number; bearing: number }
): Bookmark;

/** Remove a bookmark by id. Returns the updated array. */
export function removeBookmark(bookmarks: Bookmark[], id: string): Bookmark[];
```

**Integration with `main.ts`:**

- Add a "Save View" button after the preset grid section head, alongside the "Fast camera jumps" label. On click: prompt for a name (using `window.prompt` — keeps it dead simple, no modal needed), read current camera state from `map`, call `createBookmark`, append to array, `saveBookmarks`, re-render the bookmark cards.
- Render bookmark cards in the same `.preset-grid` container, after the built-in preset buttons. Each card has a delete affordance (small "x" button that stops propagation and calls `removeBookmark`).
- On `bootstrap()`, call `loadBookmarks()` and render any saved bookmarks into the grid.
- Clicking a bookmark card calls `map.flyTo()` with the same parameters as built-in presets.

**CSS changes:**

- `.bookmark-delete` — small absolute-positioned button inside the card, top-right corner. `opacity: 0` by default, `opacity: 1` on `.preset-card:hover .bookmark-delete`. Minimal: about 10 lines of CSS.

#### Test plan

Tests in `src/bookmarks.test.ts`. Pure functions only — mock `localStorage` via a plain object.

- `loadBookmarks()` returns `[]` when storage is empty.
- `loadBookmarks()` returns `[]` when storage contains invalid JSON.
- `saveBookmarks` then `loadBookmarks` round-trips correctly.
- `createBookmark` generates a valid bookmark with auto-caption.
- `removeBookmark` removes the correct entry and returns the rest.
- `loadBookmarks` caps at `MAX_BOOKMARKS` (truncates oldest if storage somehow exceeds).

### 3. Keyboard Navigation

#### What it does

Adds single-key shortcuts for all scene toggles, preset jumps, popup dismissal, and search focus. Shortcuts are suppressed when the search input has focus so typing doesn't trigger map actions. Toggle buttons gain `aria-pressed` for screen reader support.

#### Design

**New file:** `src/keyboardNav.ts`

**Types and constants:**

```ts
type KeyBinding = {
  key: string;
  action: () => void;
};

const TOGGLE_KEYS: Record<string, string> = {
  t: "terrain",
  n: "night",
  w: "weather",
  a: "aircraft",
  s: "ships",
  b: "buildings",
  r: "relief",
  o: "spin",
};

const PRESET_KEYS: Record<string, number> = {
  "1": 0, "2": 1, "3": 2, "4": 3, "5": 4, "6": 5,
};
```

**Key functions:**

```ts
/** Build the keydown handler. Takes callbacks so the module has no DOM/map dependency. */
export function createKeydownHandler(options: {
  isInputFocused: () => boolean;
  toggleByName: (name: string) => void;
  activatePreset: (index: number) => void;
  closePopup: () => void;
  focusSearch: () => void;
}): (event: KeyboardEvent) => void;

/** Resolve which action a key maps to: "toggle", "preset", "escape", "search", or null. */
export function resolveKeyAction(
  key: string
): { type: "toggle"; name: string } | { type: "preset"; index: number } | { type: "escape" } | { type: "search" } | null;
```

The handler calls `event.preventDefault()` only for matched keys (not for browser-reserved shortcuts like Ctrl+T).

**Integration with `main.ts`:**

- In `bootstrap()`, after all wiring: call `document.addEventListener("keydown", createKeydownHandler({ ... }))`.
- `isInputFocused`: `() => document.activeElement === searchInput`. For Escape inside the search input: blur the input instead of closing the popup.
- `toggleByName`: programmatically `.click()` the matching `[data-toggle]` button.
- `activatePreset`: programmatically `.click()` the preset button at that index in `presetButtons`.
- `closePopup`: calls `activePopup?.remove(); activePopup = null;`.
- `focusSearch`: calls `searchInput.focus()`.
- Add `aria-pressed` to all `.toggle-chip` buttons: set initial value in HTML, update in `wireToggles` click handler alongside `is-active` class toggle.

**CSS changes:** None — `aria-pressed` is a semantic attribute, not a visual one. The existing `is-active` class already handles the visual state.

#### Test plan

Tests in `src/keyboardNav.test.ts`. Tests exercise `resolveKeyAction` and `createKeydownHandler` with stub callbacks.

- `resolveKeyAction("t")` returns `{ type: "toggle", name: "terrain" }`.
- `resolveKeyAction("3")` returns `{ type: "preset", index: 2 }`.
- `resolveKeyAction("Escape")` returns `{ type: "escape" }`.
- `resolveKeyAction("/")` returns `{ type: "search" }`.
- `resolveKeyAction("x")` returns `null`.
- `resolveKeyAction("T")` (uppercase) returns the same as lowercase — case-insensitive.
- Handler does not call any callback when `isInputFocused()` returns `true` (except Escape, which should still work).
- Handler with `isInputFocused: () => false` calls `toggleByName("terrain")` when key is `"t"`.
- Modifier keys (Ctrl, Alt, Meta) suppress all bindings to avoid hijacking browser shortcuts.

### 4. Coordinate Readout

#### What it does

Shows cursor latitude, longitude, and terrain elevation in a compact bar anchored near the bottom of the map. Updates live as the mouse moves. Clicking the readout copies coordinates to the clipboard. Hidden on touch-only devices where there is no persistent cursor.

#### Design

**New file:** `src/coordinateReadout.ts`

**Types and constants:**

```ts
type CursorPosition = {
  lat: number;
  lng: number;
  elevation: number | null;  // meters, null when terrain is off or unavailable
};

const THROTTLE_MS = 100;  // ~10 Hz
```

**Key functions:**

```ts
/** Format a CursorPosition into the display string. */
export function formatReadout(pos: CursorPosition): string;
// e.g. "55.9533, -3.1883 · 142 m" or "55.9533, -3.1883" when elevation is null

/** Format coordinates for clipboard (no elevation). */
export function formatForClipboard(lat: number, lng: number): string;
// e.g. "55.9533, -3.1883"

/** Create a throttled mousemove handler. Returns the handler and a cleanup function. */
export function createReadoutController(options: {
  getElevation: (lngLat: { lng: number; lat: number }) => number | null;
  onUpdate: (text: string) => void;
}): {
  handleMouseMove: (event: { lngLat: { lng: number; lat: number } }) => void;
  dispose: () => void;
};

/** Simple leading-edge throttle. */
export function throttle<T extends (...args: never[]) => void>(fn: T, ms: number): T;
```

The throttle is a simple leading-edge throttle (fires immediately, then suppresses for `ms`). This gives responsive initial feedback without flooding the DOM.

**Integration with `main.ts`:**

- In the HTML template, add a `<div id="coord-readout" class="coord-readout" hidden></div>` after the status pill.
- In `wireMap()`, after map is ready:
  - Create the readout controller with `getElevation` calling `map.queryTerrainElevation(lngLat)` and normalizing via `normalizeTerrainElevation()`, gated on `mapState.terrainEnabled`.
  - `map.on("mousemove", controller.handleMouseMove)`.
  - On first mousemove, unhide the readout element (covers the touch-device case: no mousemove means it stays hidden).
  - Add a click listener on the readout element that calls `navigator.clipboard.writeText()` and briefly shows "Copied!" in the status pill.

**CSS changes:**

```css
.coord-readout {
  position: absolute;
  left: 50%;
  bottom: 1rem;
  transform: translateX(-50%);
  z-index: 2;
  padding: 0.5rem 0.85rem;
  border-radius: 999px;
  border: 1px solid var(--panel-border);
  background: rgba(8, 14, 24, 0.76);
  backdrop-filter: blur(14px);
  color: #edf6ff;
  font-size: 0.82rem;
  font-variant-numeric: tabular-nums;
  pointer-events: auto;
  cursor: copy;
  white-space: nowrap;
  user-select: none;
}
```

Positioned center-bottom, visually consistent with the status pill. `font-variant-numeric: tabular-nums` prevents the numbers from jittering as they update. `pointer-events: auto` so the click-to-copy works. On mobile (`max-width: 900px`), shift `bottom` up to avoid overlapping the status pill.

#### Test plan

Tests in `src/coordinateReadout.test.ts`. Pure function tests.

- `formatReadout({ lat: 55.9533, lng: -3.1883, elevation: 142 })` returns `"55.9533, -3.1883 · 142 m"`.
- `formatReadout({ lat: 0, lng: 0, elevation: null })` returns `"0.0000, 0.0000"`.
- `formatReadout` with negative coordinates formats correctly (southern/western hemispheres).
- `formatForClipboard(55.9533, -3.1883)` returns `"55.9533, -3.1883"`.
- `throttle` fires immediately on first call, suppresses subsequent calls within the window, then fires again after the window expires.
- `createReadoutController` calls `onUpdate` with the formatted string when `handleMouseMove` is invoked.
- `createReadoutController` calls `getElevation` with the lngLat from the event.
- `dispose` prevents further `onUpdate` calls.

## Implementation Order

1. **Deep-Link URL Sharing** — no dependencies on other features. Provides immediate shareability. The `parseHash` output also establishes the pattern for restoring state on load, which bookmarks will later extend.

2. **Keyboard Navigation** — independent of all other features. Small surface area, high accessibility value. Should be wired after all toggles and presets exist so the click-forwarding works naturally.

3. **Coordinate Readout** — independent. Reuses existing `normalizeTerrainElevation` and `queryTerrainElevation` patterns already in `main.ts` (`getTerrainHeight`). The throttle utility is new but trivial.

4. **Bookmarks / My Places** — last because it modifies the preset grid DOM and should be built after the hash feature is stable (bookmarks could later encode into the hash, but that is out of scope for this pass). Also the most UI-heavy of the four.

No feature depends on another at the code level. The ordering is purely about risk and review efficiency: pure-logic features first, DOM-mutation features last.

## Open Questions

1. **Hash vs. query string:** The hash approach avoids page reloads and needs no server cooperation. But if we ever want server-side rendering or OpenGraph previews, we would need to move to query parameters. For now, hash is correct — no server component exists.

2. **Bookmark limit UX:** The design caps at 24 bookmarks silently. Should we show a message when the user hits the limit, or is silent oldest-eviction acceptable? Leaning toward showing a brief status pill message.

3. **Keyboard shortcut discoverability:** There is no visible hint that shortcuts exist. Options: (a) do nothing and let power users discover them, (b) add a small "?" tooltip or section in the control dock. Recommend starting with (a) and adding discoverability later if users ask.

4. **Coordinate readout on touch devices:** The design hides the readout entirely on touch-only devices (no mousemove fires). An alternative is to show coordinates on long-press. Recommend deferring that to a follow-up — long-press conflicts with map panning and adds mobile gesture complexity.
