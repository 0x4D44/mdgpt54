## 2. Solar Terminator (Day/Night Overlay)

### Goal

Render the real-time day/night boundary on the globe as a semi-transparent dark polygon. Updates every 60 seconds.

### Approach

A new module `src/overlays/solarTerminator.ts` that:
1. Computes the subsolar point from current UTC time using simplified NOAA formulae (~60 lines of trig — no external dependencies).
2. Generates a GeoJSON polygon for the night hemisphere by computing a 90-degree spherical circle centred on the subsolar antipode (Vincenty destination formula, 180 points at 2-degree steps).
3. Adds a MapLibre `fill` layer with ~20% opacity black fill.
4. Exposes `start(map)` / `stop()` to manage the 60-second `setInterval`.

### Solar math (inlined, no library)

```
Input: Date

Julian century t = (date.getTime() - 946728000000) / 3155760000000

Mean longitude L0 = (280.46646 + 36000.76983 * t) % 360
Mean anomaly  M  = 357.52911 + 35999.05029 * t
Eccentricity  e  = 0.016708634 - 0.000042037 * t
Obliquity     ε  = 23.439291 - 0.013004167 * t

Sun equation of center C = (1.9146 - 0.004817*t)*sin(M) + 0.019993*sin(2M)
Sun true longitude      = L0 + C
Apparent longitude  λ   = true_lon - 0.00569 - 0.00478*sin(Ω)
  where Ω = 125.04 - 1934.136*t

Declination δ = asin(sin(ε) * sin(λ))

Equation of time (minutes):
  y = tan(ε/2)²
  EoT = 4*(180/π) * [y*sin(2*L0) - 2*e*sin(M) + 4*e*y*sin(M)*cos(2*L0)
                      - 0.5*y²*sin(4*L0) - 1.25*e²*sin(2*M)]

Subsolar point:
  lat = δ
  lon = -15 * (UTC_hours + UTC_minutes/60 + EoT/60 - 12)
```

### Night polygon generation

Antipode of subsolar point → `[sunLon + 180, -sunLat]`. Walk a 90-degree spherical circle around it:

```
for θ = 0 to 358 step 2:
  lat2 = asin(sin(lat1)*cos(π/2) + cos(lat1)*sin(π/2)*cos(θ))
  lon2 = lon1 + atan2(sin(θ)*sin(π/2)*cos(lat1), cos(π/2) - sin(lat1)*sin(lat2))
  → push [lon2°, lat2°]
```

This produces a valid GeoJSON Polygon (~180 coordinate pairs). No antimeridian issues because MapLibre's globe projection handles wrapping.

### Map integration

- Source: `night-overlay` (GeoJSON, initially empty FeatureCollection)
- Layer: `night-overlay` (fill, black, opacity 0.18)
- Inserted below the first label layer so labels remain readable.
- On `start(map)`: compute polygon, set source data, start 60s interval.
- On `stop()`: clear interval, remove source+layer.

### UI integration

Add a `Night` toggle chip to the Scene section in the control dock, alongside Terrain/Relief/Buildings/Spin. Default: **on**. Toggling calls `start()`/`stop()`.

## 3. Weather Radar (RainViewer)

### Goal

Overlay live precipitation radar on the map. Free, no API key.

### Data source

**RainViewer public API** — provides global composite radar as standard XYZ raster tiles.

- Metadata endpoint: `https://api.rainviewer.com/public/weather-maps.json`
- Tile URL template: `{host}{path}/512/{z}/{x}/{y}/2/1_0.png`
  - Color scheme `2` (Universal Blue), smooth `1`, snow `0`
- Max zoom: 7 (city-scale resolution)
- Updates every ~10 minutes with 2 hours of historical frames
- Coverage: ~80 countries, 1100+ radar stations. Gaps in Africa and parts of central Asia.

**Risk note:** RainViewer scaled back their API in early 2026. The tiled composites endpoint still works for personal use but has no SLA. Acceptable for a dev sandbox; if it disappears we just remove the layer.

### Approach

A new module `src/overlays/weatherRadar.ts` that:
1. Fetches `weather-maps.json` to get the latest radar timestamp and tile path.
2. Adds a MapLibre `raster` source + layer using the latest tile path.
3. Re-fetches the metadata every 5 minutes and swaps the tile URL when a new frame is available (update the source's tiles array and call `map.style.sourceCaches['weather-radar'].clearTiles(); map.triggerRepaint()`).
4. Exposes `start(map)` / `stop()`.

### Map integration

- Source: `weather-radar` (raster, tileSize 512, maxzoom 7)
- Layer: `weather-radar` (raster, opacity 0.6)
- Inserted above satellite imagery but below vector roads/labels so roads remain visible through the semi-transparent radar.

### UI integration

Add a `Weather` toggle chip to the Scene section. Default: **off** (opt-in to avoid unnecessary API calls). Toggling calls `start()`/`stop()`.

When active, show a small status note below the Scene toggles: "Radar: [timestamp] · RainViewer" (formatted as relative time like "3 min ago").

### Error handling

- If the metadata fetch fails: log a warning, set status to "Radar unavailable", leave the layer hidden. Retry on next 5-minute cycle.
- If tiles fail to load: MapLibre handles this gracefully (blank tiles). No special handling needed.

### Files

| File | Action |
|---|---|
| `src/overlays/weatherRadar.ts` | New — metadata fetch, raster source/layer management, refresh loop |
| `src/overlays/weatherRadar.test.ts` | New — metadata parsing, tile URL construction, refresh logic |
| `src/main.ts` | Add toggle chip HTML, wire toggle, call start/stop |

## 4. Road Traffic (TomTom)

### Goal

Overlay live road traffic flow as a colour-coded raster layer (green/yellow/red like Google Maps).

### Data source

**TomTom Traffic Flow Tiles** — transparent PNG raster tiles showing traffic speed relative to free-flow.

- Tile URL: `https://api.tomtom.com/traffic/map/4/tile/flow/relative0/{z}/{x}/{y}.png?tileSize=256&key={KEY}`
  - Style `relative0`: transparent where free-flow, yellow/red for congestion. Best for overlay on an existing map.
- Free tier: 50,000 tile requests/day (no credit card). Ample for personal use.
- Requires a free TomTom developer API key.

### API key handling

The key is read from the Vite environment variable `VITE_TOMTOM_API_KEY`. If absent, the Road Traffic toggle is hidden from the UI entirely (same pattern as AISStream ships key). No feature flags — just conditional UI.

### Approach

A new module `src/overlays/roadTraffic.ts` that:
1. Reads the API key from `import.meta.env.VITE_TOMTOM_API_KEY`.
2. If no key, exports `available = false` and all functions are no-ops.
3. Adds a MapLibre `raster` source + layer with the TomTom tile URL.
4. Exposes `start(map)` / `stop()` and `available: boolean`.

No refresh loop needed — MapLibre re-fetches tiles on pan/zoom automatically, and TomTom tiles are already live data (cached ~2 minutes server-side).

### Map integration

- Source: `road-traffic` (raster, tileSize 256, minzoom 6, maxzoom 18)
- Layer: `road-traffic` (raster, opacity 0.7)
- Inserted above road line layers but below labels and building extrusions.

### UI integration

Add a `Road Traffic` toggle chip to the Scene section, but **only if `available` is true**. Default: **off**. Toggling calls `start()`/`stop()`.

### Files

| File | Action |
|---|---|
| `src/overlays/roadTraffic.ts` | New — tile source, layer management, key check |
| `src/overlays/roadTraffic.test.ts` | New — key presence logic, URL construction |
| `src/main.ts` | Conditionally add toggle chip, wire toggle |

## 5. Implementation Plan

### Step order

1. **Presets** — edit `PRESETS` array. Zero risk, instant value.
2. **Solar terminator** — self-contained module, pure math, no external dependencies.
3. **Weather radar** — one external API, simple raster overlay.
4. **Road traffic** — one external API + key, simple raster overlay.

Each step is independently shippable and testable. No step depends on another.

### New file structure

```
src/overlays/
  solarTerminator.ts
  solarTerminator.test.ts
  weatherRadar.ts
  weatherRadar.test.ts
  roadTraffic.ts
  roadTraffic.test.ts
```

### Changes to existing files

Only `src/main.ts` is modified:
- `PRESETS` array updated (step 1)
- Three new toggle chips added to the Scene section HTML (steps 2-4)
- `bootstrap()` calls the overlay start/stop wiring after `wireToggles()` (steps 2-4)

### What we are NOT doing

- No animation of radar frames (just latest snapshot — animation is a future enhancement).
- No weather conditions panel (temp, wind, etc.) — radar overlay is sufficient for v1.
- No custom styling or theming changes.
- No new npm dependencies.
- No backend changes.
- No feature flags.
- No migration support.
