# HLD -- Scene Overlays and Tools

## Overview

Four new features that extend Worldviewer's scene overlay system: a time-of-day scrubber that lets users preview daylight at any datetime, an ISS live-position tracker, a USGS earthquake layer, and a click-to-measure geodesic distance tool. Each feature follows the existing enable/disable overlay lifecycle (revision tokens, load handlers, timer management) established by `solarTerminator.ts` and `weatherRadar.ts`. No new npm dependencies -- all rendering uses MapLibre native layers and GeoJSON sources.

---

## Features

### 1. Time-of-Day Scrubber

#### What it does

Adds a horizontal slider to the control dock that overrides the solar terminator's clock. Users drag to any time of day/year and see the night hemisphere update in real-time. A "Reset to now" button returns to live mode and re-enables the 60-second auto-refresh timer.

#### Design

**New files / modules**

- `src/overlays/timeScrubber.ts` -- slider state management, datetime formatting, integration hook
- `src/overlays/timeScrubber.test.ts` -- pure function tests

**Key types and functions**

```
type TimeScrubberState = {
  overrideDate: Date | null;       // null = live mode
};

// Convert slider position (0..1) to a Date for the selected year-day + time
function sliderPositionToDate(position: number, referenceYear: number): Date

// Format a Date for the slider label: "2026-03-16 14:30 UTC"
function formatScrubberLabel(date: Date): string

// Create the scrubber controller
function createTimeScrubber(options: {
  onDateChange: (date: Date | null) => void;
}): {
  mount(container: HTMLElement): void;
  unmount(): void;
  reset(): void;
}
```

**Data flow**

1. Slider emits `input` events. `sliderPositionToDate` converts the position to a `Date`.
2. `onDateChange` callback fires into `main.ts`, which stores the override date.
3. `main.ts` destroys and recreates the solar terminator overlay with `getNow: () => overrideDate` when an override is active, or with no `getNow` option when reset to live.
4. When override is active, the terminator's internal 60-second timer still fires but `getNow` returns the frozen override date, so the geometry stays stable. Alternatively, disable/re-enable with a very large `updateIntervalMs` to avoid pointless redraws.

**Integration with main.ts**

- Add the scrubber DOM below the Scene toggle grid, inside the `toggle-section`.
- When the slider changes: call `solarTerminator.disable(map)`, recreate with new `getNow`, call `solarTerminator.enable(map)`.
- "Reset to now": same pattern but with default `getNow`.
- The night toggle chip still controls whether the terminator is visible at all; the scrubber only appears when night overlay is active.

**CSS / UI changes**

- `.time-scrubber` container: `display: grid; gap: 0.4rem;`
- `.time-scrubber input[type="range"]` styled to match dark theme (custom track/thumb using accent color).
- `.time-scrubber-label` small muted text showing the selected datetime.
- `.time-scrubber-reset` pill button matching `.toggle-chip` style.
- Scrubber hidden when night overlay is disabled.

#### Test plan

- `sliderPositionToDate`: position 0 = midnight Jan 1, position 0.5 = ~Jul 2 noon, position 1 = Dec 31 23:59. Verify output is a valid Date in the correct year.
- `formatScrubberLabel`: spot-check known dates produce expected "YYYY-MM-DD HH:MM UTC" strings.
- Integration: mount/unmount does not leak DOM nodes. `reset()` calls `onDateChange(null)`.

### 2. ISS Live Position

#### What it does

Polls the Where The ISS At API every 5 seconds and renders the International Space Station as a dedicated symbol on the globe. A faded trail of the last ~60 positions (5 minutes) follows the icon. The overlay is only visible at globe zoom levels (z < 5) and fades out as the user zooms in. Toggled via a chip in the Scene section.

#### Design

**New files / modules**

- `src/overlays/issTracker.ts` -- overlay lifecycle, fetch, GeoJSON construction
- `src/overlays/issTracker.test.ts` -- pure function tests, MockMap lifecycle tests

**Key types and functions**

```
const ISS_API_URL = "https://api.wheretheiss.at/v1/satellites/25544";
const ISS_POLL_INTERVAL_MS = 5_000;
const ISS_TRAIL_MAX_POINTS = 60;
const ISS_SOURCE_ID = "iss-position";
const ISS_TRAIL_SOURCE_ID = "iss-trail";
const ISS_ICON_LAYER_ID = "iss-icon";
const ISS_TRAIL_LAYER_ID = "iss-trail-line";

type IssPosition = {
  latitude: number;
  longitude: number;
  altitude: number;     // km
  velocity: number;     // km/h
  timestamp: number;    // unix seconds
};

// Parse API JSON response into typed position, returning null on bad data
function parseIssResponse(data: unknown): IssPosition | null

// Build GeoJSON Point feature for the ISS icon
function buildIssFeature(position: IssPosition): GeoJSON.Feature<GeoJSON.Point>

// Build GeoJSON LineString from trail positions, splitting at the antimeridian
function buildIssTrailFeature(trail: IssPosition[]): GeoJSON.Feature<GeoJSON.MultiLineString>

// Overlay lifecycle (same shape as solarTerminator/weatherRadar)
function createIssTrackerOverlay(options?: {
  fetchImpl?: typeof fetch;
  pollIntervalMs?: number;
  onStateChange?: (presentation: { note: string | null }) => void;
}): {
  enable(map: Map): void;
  disable(map: Map): void;
}
```

**MapLibre layer specs**

ISS icon layer:
```
{
  id: "iss-icon",
  type: "circle",
  source: "iss-position",
  maxzoom: 5,
  paint: {
    "circle-radius": 6,
    "circle-color": "#ffffff",
    "circle-stroke-color": "#67d0ff",
    "circle-stroke-width": 2,
    "circle-opacity": ["interpolate", ["linear"], ["zoom"], 3, 1, 5, 0]
  }
}
```

ISS trail layer:
```
{
  id: "iss-trail-line",
  type: "line",
  source: "iss-trail",
  maxzoom: 5,
  paint: {
    "line-color": "#67d0ff",
    "line-width": 1.5,
    "line-opacity": ["interpolate", ["linear"], ["zoom"], 3, 0.4, 5, 0]
  }
}
```

**Data flow**

1. `enable` adds sources and layers, starts polling with `setInterval`.
2. Each poll: fetch ISS API, parse response, append to trail ring buffer (capped at 60 entries), rebuild both GeoJSON features, call `source.setData()`.
3. `disable` clears interval, aborts in-flight fetch, removes layers and sources.
4. Revision token pattern guards against stale callbacks, identical to solarTerminator.

**Layer ordering**: insert both layers before the first label layer (same anchor logic as solarTerminator).

**Integration with main.ts**

- Add `issEnabled: boolean` to `MapState` (default `false`).
- Add `<button type="button" class="toggle-chip" data-toggle="iss">ISS</button>` to the toggle grid.
- Wire toggle case in `wireToggles`: flip state, toggle class, call `syncSceneOverlays`.
- In `syncSceneOverlays`: `issEnabled ? issTracker.enable(map) : issTracker.disable(map)`.
- Feed `onStateChange` notes into `renderSceneOverlayPresentation` (show altitude/velocity when active).

**CSS / UI changes**

None beyond the toggle chip (existing styles apply).

#### Test plan

- `parseIssResponse`: valid JSON returns typed `IssPosition`; missing/malformed fields return `null`.
- `buildIssFeature`: returns Point with correct coordinates and properties.
- `buildIssTrailFeature`: empty trail returns empty MultiLineString. Trail crossing antimeridian splits into separate line segments.
- Lifecycle with MockMap: enable adds source+layer, disable removes them. Rapid enable/disable does not leak timers. Enable while style not loaded defers via load handler.

### 3. Earthquake Layer

#### What it does

Fetches the USGS 2.5+ magnitude earthquake feed (updated every 5 minutes) and renders quakes as circles on the map, sized and colored by magnitude. Clicking a circle shows a popup with magnitude, location, depth, and event time. Toggled via a chip in the Scene section.

#### Design

**New files / modules**

- `src/overlays/earthquakeLayer.ts` -- overlay lifecycle, fetch, popup wiring
- `src/overlays/earthquakeLayer.test.ts` -- pure function tests, MockMap lifecycle tests

**Key types and functions**

```
const EARTHQUAKE_API_URL =
  "https://earthquake.usgs.gov/earthquakes/feed/v1.0/summary/2.5_day.geojson";
const EARTHQUAKE_REFRESH_INTERVAL_MS = 300_000;
const EARTHQUAKE_SOURCE_ID = "earthquakes";
const EARTHQUAKE_LAYER_ID = "earthquake-circles";

type EarthquakeProperties = {
  mag: number;
  place: string;
  time: number;       // unix ms
  depth: number;      // km (from geometry coordinates[2])
};

// The USGS feed is already a FeatureCollection -- no parsing needed.
// This function validates the response shape and extracts depth into properties.
function normalizeEarthquakeFeatures(
  fc: GeoJSON.FeatureCollection
): GeoJSON.FeatureCollection

// Format earthquake popup content
function formatEarthquakePopup(props: EarthquakeProperties): string

// Magnitude -> circle radius (px), clamped
function earthquakeRadius(magnitude: number): number

// Magnitude -> fill color hex string
function earthquakeColor(magnitude: number): string

// Overlay lifecycle
function createEarthquakeOverlay(options?: {
  fetchImpl?: typeof fetch;
  updateIntervalMs?: number;
  onStateChange?: (presentation: { note: string | null; creditLabel: string | null }) => void;
}): {
  enable(map: Map): void;
  disable(map: Map): void;
}
```

**MapLibre layer spec**

```
{
  id: "earthquake-circles",
  type: "circle",
  source: "earthquakes",
  paint: {
    "circle-radius": [
      "interpolate", ["linear"], ["get", "mag"],
      2.5, 4,
      5.0, 10,
      7.0, 20,
      9.0, 30
    ],
    "circle-color": [
      "interpolate", ["linear"], ["get", "mag"],
      2.5, "#fef0d9",
      4.0, "#fdcc8a",
      5.5, "#fc8d59",
      7.0, "#e34a33",
      8.5, "#b30000"
    ],
    "circle-opacity": 0.8,
    "circle-stroke-color": "#ffffff",
    "circle-stroke-width": 1
  }
}
```

**Data flow**

1. `enable` adds GeoJSON source (empty FeatureCollection), adds circle layer, fetches USGS feed, calls `source.setData(fc)`, starts 5-minute refresh timer.
2. On click: query rendered features on `earthquake-circles`, extract properties, build popup HTML with `formatEarthquakePopup`, show MapLibre Popup. Pattern follows `showTrafficPopup` from `trafficLayers.ts`.
3. `disable` clears timer, aborts fetch, removes layer/source, removes any open earthquake popup.
4. Revision token + abort controller pattern identical to weatherRadar.

**Layer ordering**: insert before first label layer (same as other overlays).

**Popup**: uses existing `.popup-card` CSS classes. Kicker = "Earthquake", title = "M{mag} -- {place}", meta = "{depth} km deep | {formatted time}".

**Integration with main.ts**

- Add `earthquakeEnabled: boolean` to `MapState` (default `false`).
- Add `<button type="button" class="toggle-chip" data-toggle="earthquakes">Quakes</button>` to the toggle grid.
- Wire toggle case: flip state, toggle class, call `syncSceneOverlays`.
- In `syncSceneOverlays`: `earthquakeEnabled ? earthquakeOverlay.enable(map) : earthquakeOverlay.disable(map)`.
- Wire click handler inside the overlay's `enable` method (not in main.ts) to keep popup logic self-contained. Use a module-level `Popup` reference cleared on disable, same pattern as `trafficPopup` in `trafficLayers.ts`.

**CSS / UI changes**

None beyond the toggle chip. Popup reuses existing `.popup-card` styles.

#### Test plan

- `normalizeEarthquakeFeatures`: valid FeatureCollection passes through with depth added to properties. Empty/malformed input returns empty FeatureCollection.
- `earthquakeRadius`: known magnitudes produce expected pixel radii. Edge cases: negative magnitude, NaN.
- `earthquakeColor`: spot-check magnitude breakpoints.
- `formatEarthquakePopup`: produces expected HTML string with escaped text.
- Lifecycle with MockMap: enable/disable adds/removes source+layer. Timer cleared on disable. Fetch abort on rapid re-enable.

### 4. Measurement Tool

#### What it does

Click-to-measure mode: the user clicks two points on the map, and a geodesic great-circle line appears between them with distance (km/mi) and initial bearing displayed in a compact readout. Escape exits measurement mode and clears the line. A "Measure" toggle chip in the Scene section activates the mode.

#### Design

**New files / modules**

- `src/overlays/measureTool.ts` -- tool state machine, map event wiring, GeoJSON construction
- `src/overlays/measureGeodesic.ts` -- pure geodesic math (distance, bearing, intermediate points)
- `src/overlays/measureGeodesic.test.ts` -- pure function tests for geodesic math
- `src/overlays/measureTool.test.ts` -- state machine and MockMap tests

**Key types and functions**

`measureGeodesic.ts` (pure, no MapLibre dependency):
```
type LngLat = { lng: number; lat: number };

// Haversine great-circle distance in meters
function geodesicDistanceMeters(a: LngLat, b: LngLat): number

// Initial bearing in degrees (0-360, clockwise from north)
function geodesicBearing(from: LngLat, to: LngLat): number

// Generate N intermediate points along the great circle for smooth rendering
function geodesicIntermediatePoints(a: LngLat, b: LngLat, segments: number): LngLat[]

// Format distance for display: "1,234 km (767 mi)"
function formatDistance(meters: number): string

// Format bearing for display: "045.2 deg"
function formatBearing(degrees: number): string
```

`measureTool.ts`:
```
const MEASURE_SOURCE_ID = "measure-tool";
const MEASURE_LINE_LAYER_ID = "measure-line";
const MEASURE_POINTS_LAYER_ID = "measure-points";

type MeasureState = "idle" | "first-click" | "complete";

type MeasureResult = {
  distanceMeters: number;
  bearingDegrees: number;
  from: LngLat;
  to: LngLat;
};

function createMeasureTool(options?: {
  geodesicSegments?: number;   // default 64
  onStateChange?: (state: MeasureState, result: MeasureResult | null) => void;
}): {
  enable(map: Map): void;
  disable(map: Map): void;
}
```

**MapLibre layer specs**

Measurement line:
```
{
  id: "measure-line",
  type: "line",
  source: "measure-tool",
  filter: ["==", ["geometry-type"], "LineString"],
  paint: {
    "line-color": "#f4c989",
    "line-width": 2.5,
    "line-dasharray": [4, 3]
  }
}
```

Measurement points:
```
{
  id: "measure-points",
  type: "circle",
  source: "measure-tool",
  filter: ["==", ["geometry-type"], "Point"],
  paint: {
    "circle-radius": 5,
    "circle-color": "#f4c989",
    "circle-stroke-color": "#ffffff",
    "circle-stroke-width": 1.5
  }
}
```

**State machine**

```
idle -> (click) -> first-click     [show point A]
first-click -> (click) -> complete [show point B + line + readout]
complete -> (click) -> first-click  [clear, start new measurement from click]
any -> (Escape) -> idle             [clear everything]
any -> (disable) -> idle            [clear everything]
```

**Data flow**

1. `enable` adds a GeoJSON source (empty FeatureCollection), adds line and point layers, sets cursor to `crosshair`, attaches `click` and `keydown` listeners on the map/document.
2. First click: store point A, update source with one Point feature.
3. Second click: store point B, compute `geodesicIntermediatePoints` to build a smooth LineString, compute distance and bearing, update source with two Points + one LineString, fire `onStateChange` with `MeasureResult`.
4. `onStateChange` callback in `main.ts` updates a readout element below the toggle grid (or repurposes the `scene-overlay-note` element).
5. Escape: clear source data, reset state to idle, fire `onStateChange(idle, null)`.
6. `disable` removes listeners, removes layers and source, resets cursor.

**Integration with main.ts**

- Add `measureEnabled: boolean` to `MapState` (default `false`).
- Add `<button type="button" class="toggle-chip" data-toggle="measure">Measure</button>` to the toggle grid.
- Wire toggle case: flip state, toggle class, call `syncSceneOverlays`.
- In `syncSceneOverlays`: `measureEnabled ? measureTool.enable(map) : measureTool.disable(map)`.
- `onStateChange` callback: when state is `complete`, show formatted distance and bearing in `scene-overlay-note`. When state is `idle`, clear it.
- Measurement mode is mutually exclusive with map click popups -- while measure is active, suppress the default feature-click popup in `wireMap`.

**CSS / UI changes**

- Map cursor set to `crosshair` via `map.getCanvas().style.cursor` while measure mode is active.
- No new CSS classes needed; readout uses existing `scene-overlay-note`.

#### Test plan

- `geodesicDistanceMeters`: London-to-New York ~5,570 km. Same point = 0. Antipodal points ~ 20,015 km.
- `geodesicBearing`: due east from equator/prime-meridian to equator/90E = 90 deg. Due north = 0 deg.
- `geodesicIntermediatePoints`: 2 segments returns 3 points (including endpoints). Midpoint of equatorial pair lies on equator.
- `formatDistance`: 1000 m -> "1.0 km (0.6 mi)". 100 m -> "100 m (328 ft)".
- `formatBearing`: 0 -> "000.0 deg", 359.9 -> "359.9 deg".
- State machine: idle -> click -> first-click -> click -> complete. Escape from any state -> idle. Double-enable is idempotent. Disable removes all map artifacts.
- MockMap lifecycle: enable/disable adds/removes source and layers. Click handlers detached on disable.

## Implementation Order

1. **Measurement Tool** -- no external API dependency, pure math is easily testable, immediately useful. Ship `measureGeodesic.ts` with full tests first, then `measureTool.ts`.
2. **Earthquake Layer** -- simplest API integration (USGS returns ready-made GeoJSON), familiar overlay lifecycle, good popup pattern exercise before ISS.
3. **ISS Tracker** -- polling pattern, trail ring buffer, antimeridian splitting. Slightly more complex than earthquakes but same lifecycle shape.
4. **Time-of-Day Scrubber** -- depends on solar terminator already working; involves DOM slider and recreating the overlay, so it benefits from all prior overlay work being stable.

Each feature is independent and can ship behind its own toggle chip with no coupling to the others.

## Open Questions

1. **Scrubber granularity**: should the slider cover one day (24h) with a separate date picker, or a full year on a single axis? A single-axis full-year slider is simpler but loses time-of-day precision. Proposed: two sliders (date + time), or one slider for 24h and a date input field.
2. **ISS icon**: a plain circle works but a custom icon (small ISS silhouette rendered to canvas, same pattern as aircraft icons) would be more recognizable. Worth the extra code?
3. **Earthquake popup conflict**: if both earthquake and traffic popups are active, clicking an earthquake near a traffic track could fire both handlers. Should we add a `stopPropagation` pattern, or accept that the topmost layer wins (MapLibre's default)?
4. **Measure tool and touch devices**: two-tap measurement works on mobile but conflicts with double-tap-to-zoom. Should we require a long-press for the second point on touch, or just accept the conflict and let users use the toggle to exit?
5. **ISS trail antimeridian**: the trail LineString needs splitting when the ISS crosses 180/-180 longitude. The `buildIssTrailFeature` function handles this, but should we also split at large latitude jumps (polar crossings) for visual cleanliness?
