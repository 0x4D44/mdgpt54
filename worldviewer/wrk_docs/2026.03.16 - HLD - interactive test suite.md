# HLD: Interactive Test Suite

## Problem

The app has 39 Vitest unit test files covering pure logic, but nothing that exercises the assembled application in a real browser. Bugs in wiring, DOM structure, event handling, WebSocket lifecycle, and URL state only surface through manual clicking. We need an automated suite that launches the app and interacts with it like a user.

## Framework Choice: Playwright

**Why Playwright over alternatives:**

| Concern | Playwright | Cypress |
|---|---|---|
| WebSocket interception | Native `page.routeWebSocket()` | Requires workarounds |
| Multi-browser | Chromium, Firefox, WebKit | Chromium only (free tier) |
| TypeScript support | First-class | Bolt-on |
| Network mocking | `page.route()` with glob patterns | `cy.intercept()` — similar |
| Parallel execution | Built-in worker isolation | Requires paid dashboard |
| Already in stack | Vitest for units, Playwright for E2E — clean separation | Would overlap with Vitest |

Playwright is the industry standard for E2E web testing in 2026. It handles our specific needs well: WebSocket mocking for the traffic relay, network interception for tile servers and APIs, and works with WebGL canvases (though we won't screenshot-compare them).

## What to Test

The app has two categories of behaviour: **DOM-driven** (buttons, text, attributes, visibility) and **WebGL-driven** (map rendering, 3D models, overlays). DOM-driven behaviour is reliably testable. WebGL rendering varies across GPUs and is not worth screenshot-comparing in CI.

### Test Suites

#### 1. App Bootstrap (`app-bootstrap.spec.ts`)
- App shell renders: `#map`, `#control-dock`, `#status-pill` all present
- Status pill transitions from "Loading..." to ready state
- Default toggle states match expectations (Terrain=on, Relief=on, Night=on, Weather=off, etc.)
- MapLibre map instance is created (canvas element appears inside `#map`)

#### 2. Side Panel & Dock Toggle (`dock-toggle.spec.ts`)
- Panel starts expanded (`aria-expanded="true"`)
- Click toggle → panel collapses (`is-collapsed` class, `aria-expanded="false"`, button text changes `<` → `>`)
- Click again → panel re-expands

#### 3. Scene Toggle Chips (`scene-toggles.spec.ts`)
- Each toggle chip click flips `aria-pressed` and `is-active` class
- Status pill updates with the correct message for each toggle (e.g., "Terrain enabled." / "Terrain flattened.")
- Night toggle controls time scrubber visibility (`#time-scrubber` hidden attribute)
- Toggling night off resets time scrubber to live

#### 4. Search (`search.spec.ts`)
- Mock Nominatim API via `page.route('**/nominatim**')`
- Submit search → results appear in `#search-results`
- Click a result → map flies to location (verify `flyTo` called or `moveend` fires)
- Empty/error responses show appropriate message in `#search-message`
- Search input keyboard shortcut (`/`) focuses the input

#### 5. Presets & Bookmarks (`presets-bookmarks.spec.ts`)
- Click a preset card → status pill shows "Flying to {label}..."
- Keyboard shortcut (1-6) activates corresponding preset
- Save View button → prompt dialog → bookmark card appears in grid
- Bookmark card click → flies to saved location
- Bookmark delete button removes the card
- Bookmarks persist across page reload (localStorage)
- Bookmark limit (MAX_BOOKMARKS) enforced with status pill message

#### 6. URL Hash State (`url-hash.spec.ts`)
- Navigate to `/#lat=51.75&lng=-1.26&z=15&p=72&b=30` → map initializes at those coordinates
- Toggle states encoded in hash: `terrain=0` → terrain toggle starts inactive
- Camera movement updates the URL hash (after debounce)
- Hash with all defaults produces clean URL (no hash fragment)

#### 7. Traffic System (`traffic.spec.ts`)
- Mock WebSocket via `page.routeWebSocket('**/traffic')`
- Traffic section renders in the control dock (Aircraft/Ships toggle buttons)
- Send mock snapshot through WebSocket → traffic status updates
- Toggle aircraft off → `clearAircraftData` clears the map source
- Toggle ships off → `clearShipsData` clears the map source
- Connection status indicator updates on connect/disconnect/reconnect
- Layer availability reflects snapshot status (disabled when provider reports error)

#### 8. Keyboard Navigation (`keyboard-nav.spec.ts`)
- `/` focuses search input
- `Escape` blurs search and dismisses popup
- Number keys (1-6) trigger presets
- Letter shortcuts toggle scene overlays (verify `aria-pressed` flips)

#### 9. Coordinate Readout (`coord-readout.spec.ts`)
- `#coord-readout` starts hidden
- Mouse move over map canvas reveals it
- Click on readout copies coordinates to clipboard (verify via `navigator.clipboard` mock)

#### 10. Telemetry Metrics (`telemetry.spec.ts`)
- Metric elements (`#metric-mode`, `#metric-zoom`, etc.) update from "Loading..." to real values after map loads
- Mode switches between "Globe" and "Mercator" based on zoom level

## Architecture

```
e2e/
  fixtures/
    nominatim-response.json      # Canned search results
    traffic-snapshot.json        # Canned aircraft + ships snapshot
  helpers/
    mock-apis.ts                 # Reusable page.route() setup for tile servers + APIs
    mock-traffic.ts              # WebSocket mock helper for traffic relay
    wait-for-map.ts              # Wait for map canvas + idle state
  app-bootstrap.spec.ts
  dock-toggle.spec.ts
  scene-toggles.spec.ts
  search.spec.ts
  presets-bookmarks.spec.ts
  url-hash.spec.ts
  traffic.spec.ts
  keyboard-nav.spec.ts
  coord-readout.spec.ts
  telemetry.spec.ts
  playwright.config.ts
```

### Key Helper: `mock-apis.ts`

Every test needs external APIs mocked so tests don't hit the network and don't flake. This helper intercepts:

- **Tile requests** (`**/tile/**`, `*.pbf`, `*.png` terrain tiles): Return empty/stub responses. The map won't render tiles visually, but MapLibre won't error out.
- **Nominatim** (`**/search?**`): Return canned JSON from fixtures.
- **ISS tracker** (`**/iss-now**`): Return fixed position.
- **USGS earthquakes** (`**/earthquake.usgs.gov/**`): Return canned GeoJSON.
- **Weather radar** (`**/rainviewer**`): Return stub.

### Key Helper: `mock-traffic.ts`

Wraps `page.routeWebSocket()` to:
- Intercept the `/traffic` WebSocket upgrade
- Provide a `sendSnapshot(data)` method to push mock traffic data
- Provide a `close()` method to simulate disconnect
- Simulate the server-side subscribe/snapshot protocol

### Key Helper: `wait-for-map.ts`

MapLibre initialization is async. This helper:
- Waits for `canvas` element inside `#map`
- Optionally waits for status pill text to change from "Loading..."
- Returns when the app is ready for interaction

### Playwright Config

```typescript
// e2e/playwright.config.ts
import { defineConfig } from "@playwright/test";

export default defineConfig({
  testDir: ".",
  timeout: 30_000,
  retries: 1,
  use: {
    baseURL: "http://localhost:5173",
    headless: true,
  },
  webServer: {
    command: "npm run dev",
    port: 5173,
    reuseExistingServer: true,
  },
});
```

### npm Scripts

```json
{
  "test:e2e": "playwright test --config e2e/playwright.config.ts",
  "test:e2e:ui": "playwright test --config e2e/playwright.config.ts --ui"
}
```

The `--ui` flag launches Playwright's interactive test runner with time-travel debugging — useful for exploratory investigation when a test fails.

## What We Explicitly Skip

- **Visual regression / screenshot comparison**: WebGL rendering varies by GPU, driver, and OS. Not worth the flake rate. The unit tests already cover overlay geometry math.
- **Performance testing**: Not a correctness concern for this suite.
- **Mobile viewports**: The app doesn't have responsive breakpoints yet.
- **Multi-browser matrix in CI**: Run Chromium only to keep things fast. Playwright supports Firefox/WebKit if needed later.

## Dependencies to Add

```
npm install -D @playwright/test

```

## Estimated Scope

~10 spec files, ~50-70 test cases. Three shared helpers. One config file. No changes to application code needed — all tests interact through the public DOM surface.
