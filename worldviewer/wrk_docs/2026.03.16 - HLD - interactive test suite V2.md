# HLD: Interactive Test Suite (V2)

## Problem

The app has 39 Vitest unit test files covering pure logic, but nothing that exercises the assembled application in a real browser. Bugs in wiring, DOM structure, event handling, WebSocket lifecycle, and URL state only surface through manual clicking. We need an automated suite that launches the app and interacts with it like a user.

## Framework Choice: Playwright

**Why Playwright over alternatives:**

| Concern | Playwright | Cypress |
|---|---|---|
| WebSocket interception | Native `page.routeWebSocket()` | Requires workarounds |
| Multi-browser | Chromium, Firefox, WebKit | Chromium only (free tier) |
| TypeScript support | First-class | Bolt-on |
| Network mocking | `page.route()` with glob patterns | `cy.intercept()` — similar |
| Parallel execution | Built-in worker isolation | Requires paid dashboard |
| Already in stack | Vitest for units, Playwright for E2E — clean separation | Would overlap with Vitest |

**Constraint:** `page.routeWebSocket()` is Chromium-only. The suite targets Chromium only. If Firefox/WebKit are added later, the traffic WebSocket tests must be skipped or refactored.

**Minimum version:** `@playwright/test >= 1.48` (for stable `page.routeWebSocket()`).

## What to Test

The app has two categories of behaviour: **DOM-driven** (buttons, text, attributes, visibility) and **WebGL-driven** (map rendering, 3D models, overlays). DOM-driven behaviour is reliably testable. WebGL rendering varies across GPUs and is not worth screenshot-comparing in CI.

### Test Suites

#### 1. App Bootstrap (`app-bootstrap.spec.ts`)
- App shell renders: `#map`, `#control-dock`, `#status-pill` all present
- Status pill transitions from "Loading open Earth layers..." to a ready-state message
- Default toggle states match expectations (Terrain=on, Relief=on, Night=on, Weather=off, Quakes=off, ISS=off, Buildings=on, Spin=on, Measure=off)
- Canvas element appears inside `#map` (smoke test that MapLibre initialized)
- Time scrubber (`#time-scrubber`) is visible after bootstrap (because Night defaults to on)
- **Error path:** Mock OpenFreeMap style fetch to return 500 → status pill shows error message with `is-error` class

#### 2. Side Panel & Dock Toggle (`dock-toggle.spec.ts`)
- `#dock-toggle` button starts with `aria-expanded="true"`
- Click `#dock-toggle` → `#control-dock` gains `is-collapsed` class, button gets `aria-expanded="false"`, button text changes `<` → `>`
- Click again → panel re-expands, attributes revert

#### 3. Scene Toggle Chips (`scene-toggles.spec.ts`)
- Each toggle chip (`[data-toggle="..."]`) click flips `aria-pressed` and `is-active` class
- Status pill updates with the correct message for each toggle:
  - terrain: "Terrain enabled." / "Terrain flattened."
  - relief: "Relief overlay enabled." / "Relief overlay hidden."
  - night: "Night overlay enabled." / "Night overlay hidden."
  - weather: "Weather radar enabled." / "Weather radar hidden."
  - earthquakes: "Earthquake layer enabled." / "Earthquake layer hidden."
  - iss: "ISS tracker enabled." / "ISS tracker hidden."
  - buildings: "3D buildings enabled." / "Buildings hidden."
  - spin: "Orbital spin enabled." / "Orbital spin paused."
  - measure: "Measure mode: click two points to measure distance." / "Measure mode off."
- Night toggle controls time scrubber visibility (`#time-scrubber` `hidden` attribute)
- Toggling night off resets time scrubber to live

#### 4. Search (`search.spec.ts`)
- Mock Nominatim API via `page.route('**/nominatim.openstreetmap.org/**')` (same pattern as the `mock-apis` helper default; override the handler in the test)
- Fixture uses **GeocodJSON** format: `features[].properties.geocoding.label` and `features[].geometry.coordinates`
- Submit search → results appear in `#search-results`
- Click a result → status pill shows "Flying to {label}..." (verifies flyTo was triggered without waiting for animation to complete)
- Empty results → appropriate message in `#search-message`
- **Error path:** Mock Nominatim to return 500 → error message shown
- Search input keyboard shortcut (`/`) focuses the input (also covered in keyboard-nav)

#### 5. Presets & Bookmarks (`presets-bookmarks.spec.ts`)
- Click a preset card (`[data-preset="..."]`) → status pill shows "Flying to {label}..."
- Keyboard shortcut (1-6) activates corresponding preset (status pill changes)
- Save View: register `page.on('dialog', d => d.accept('My Place'))` **before** clicking `#save-view-btn` → bookmark card with `[data-bookmark-id]` appears in `.preset-grid`
- Bookmark card click → status pill shows "Flying to My Place..."
- Bookmark delete button (`.bookmark-delete`) removes the card
- Bookmarks persist: save a bookmark, reload page, verify card still present (must be within a single test — same BrowserContext = same localStorage)
- Bookmark limit: save `MAX_BOOKMARKS` bookmarks, attempt one more → status pill shows limit message

#### 6. URL Hash State (`url-hash.spec.ts`)
- Navigate to `/#lat=51.75&lng=-1.26&z=15&p=72&b=30` → verify toggle states and map initialized (check metric elements show values, not "Loading...")
- Toggle states encoded in hash: `terrain=0` → terrain toggle chip lacks `is-active` class and has `aria-pressed="false"`
- Camera movement updates the URL hash — use `page.waitForFunction(() => location.hash.includes('lat='))` after triggering a camera move (accounts for 400ms debounce)
- Hash with all defaults produces clean URL (no hash fragment)

#### 7. Traffic System (`traffic.spec.ts`)

The traffic system uses **two transports**: HTTP polling to OpenSky for aircraft, WebSocket relay for ships. Tests must mock both.

**Setup prerequisite:** Navigate to a URL with `#z=8` (or higher). Traffic activation is gated on `MIN_LIVE_TRAFFIC_ZOOM = 5`. At the default zoom of 1.2, neither aircraft polling nor the ship WebSocket connects — mocks would never be called.

- Traffic section renders in control dock — wait for `[data-traffic-toggle="aircraft"]` to appear (traffic UI is injected async after map style loads)
- Aircraft toggle and ships toggle use `[data-traffic-toggle="aircraft"]` and `[data-traffic-toggle="ships"]` selectors (NOT `[data-toggle]`)
- Mock OpenSky HTTP via `page.route('**/opensky-network.org/**')` → return canned aircraft state vectors
- Mock ship relay via `page.routeWebSocket('**/traffic')` → push mock ship data
- Toggle aircraft ON → wait for mock OpenSky data to be served → toggle aircraft OFF → verify `[data-traffic-toggle="aircraft"]` transitions to `aria-pressed="false"` with `is-active` class removed (must test the transition, not just the default off-state)
- Toggle ships ON → wait for mock ship relay data → toggle ships OFF → verify `[data-traffic-toggle="ships"]` transitions to `aria-pressed="false"` with `is-active` class removed
- When both toggles off → `#traffic-status` text reflects disconnected/off state
- Connection status indicator updates on WebSocket connect/disconnect
- **Error path:** Mock OpenSky to return 429 (rate limited) → aircraft status shows degraded hint

#### 8. Keyboard Navigation (`keyboard-nav.spec.ts`)
- **Precondition:** Ensure no element is focused before testing shortcuts (`await page.locator('body').click()`)
- `/` focuses `#search-input`
- `Escape` blurs search input
- Number keys (1-6) trigger corresponding presets (verify status pill shows "Flying to {label}...")
- Letter shortcuts toggle overlays (e.g., `t` toggles terrain — verify `aria-pressed` flips)

#### 9. Coordinate Readout (`coord-readout.spec.ts`)

**Setup:** Grant clipboard permissions via `context.grantPermissions(['clipboard-write'])` so `navigator.clipboard.writeText()` resolves (rather than rejecting, which would prevent the "Copied!" callback).

- `#coord-readout` starts `hidden`
- `page.mouse.move(x, y)` over map canvas area → readout becomes visible with coordinate text (assert *some* text appears, not specific coordinate values — MapLibre projection determines the actual numbers)
- Click on `#coord-readout` → status pill shows "Copied!"

#### 10. Telemetry Metrics (`telemetry.spec.ts`)
- After map loads, `#metric-mode` shows a zoom-bracket label (one of: "Orbit", "Continental", "Regional", "Metro", "Street") — NOT "Globe"/"Mercator"
- `#metric-zoom`, `#metric-altitude`, `#metric-pitch` show numeric values (not "--" or "Loading...")
- `#metric-terrain` shows "--" when terrain is enabled (mock tiles provide no elevation data, so `queryTerrainElevation` returns null), and "Off" when terrain is disabled. The test should assert the text is NOT "Off" when terrain is on — not that it shows a numeric elevation.

### Key Helper: `mock-apis.ts`

Every test needs external APIs mocked so tests don't hit the network and don't flake. This helper intercepts ALL HTTP requests to external services:

- **OpenFreeMap style** (`**/tiles.openfreemap.org/styles/**`): Return a minimal valid MapLibre style JSON. **This is critical** — `buildMapStyle()` fetches this before creating the map. Without it, bootstrap throws and no test can run. **The stub must include at least one layer** to avoid `undefined` in the processed layer array:
  ```json
  {
    "version": 8,
    "sources": {},
    "layers": [{ "id": "background", "type": "background", "paint": { "background-color": "#000" } }],
    "glyphs": "https://stub.local/fonts/{fontstack}/{range}.pbf"
  }
  ```
  The `glyphs` URL is intercepted by the tile pattern below, preventing MapLibre from erroring on missing font stacks.
- **Tile requests** (`*.pbf`, `*.png`, `*.jpg`): Return empty 200 responses. This covers vector tiles, terrain PNGs, satellite JPGs from EOX Maps, and font glyphs.
- **Nominatim** (`**/nominatim.openstreetmap.org/**`): Default stub returning empty results; individual tests override this handler with fixture data using the same pattern.
- **OpenSky** (`**/opensky-network.org/**`): Return canned aircraft state vectors from fixture. This covers both the states API (`/api/states/all`) and the flight route API (`/api/routes`). Without this, the browser makes direct HTTP requests to OpenSky.
- **Aircraft identity CSV shards** (`**/aircraft-identity/**`): Return empty CSV responses.
- **ISS tracker** (`**/api.wheretheiss.at/**`): Return fixed position JSON.
- **USGS earthquakes** (`**/earthquake.usgs.gov/**`): Return canned GeoJSON feature collection.
- **Weather radar** (`**/rainviewer**`): Return stub. Note: this means enabling the weather toggle will produce an error/unavailable presentation state, not rendered tiles. This is acceptable since weather rendering is out of scope.

### Key Helper: `mock-ship-relay.ts`

Wraps `page.routeWebSocket()` to handle the **ship-only** WebSocket relay:
- Intercepts the `/traffic` WebSocket upgrade
- Provides a `sendShipData(data)` method to push mock ship messages
- Provides a `close()` method to simulate disconnect
- Simulates the server-side subscribe → ship data protocol

This helper covers ships only. Aircraft arrive via HTTP (mocked in `mock-apis.ts`).

### Key Helper: `wait-for-map.ts`

MapLibre initialization is async. This helper:
- Waits for `canvas` element inside `#map`
- Waits for `#status-pill` text to NOT contain "Loading open Earth layers..." (the exact initial text)
- Uses MapLibre's `load` event semantics, **NOT `idle`** — terrain/contour tiles use a custom DEM protocol that `page.route()` cannot intercept, so tiles may fail to load and `idle` may never fire

Note: the status pill transition may be near-instant if the mocked style fetch returns quickly. This is fine — Playwright's `expect().not.toHaveText()` auto-retries and handles already-satisfied conditions.

### Playwright Config

```typescript
// e2e/playwright.config.ts
import { defineConfig } from "@playwright/test";

export default defineConfig({
  testDir: ".",
  testMatch: "*.spec.ts",        // Exclude helper/fixture .ts files
  timeout: 30_000,
  retries: 1,
  use: {
    baseURL: "http://localhost:5173",
    headless: true,
  },
  webServer: {
    command: "npm run dev:web",   // Vite only — no traffic relay needed
    port: 5173,
    reuseExistingServer: true,
  },
});
```

**Why `dev:web` not `dev`:** The `dev` script starts both Vite and the traffic relay via `concurrently`. The relay needs an `AISSTREAM_API_KEY` and binds port 3210. Since all traffic connections are mocked by Playwright at the browser level, the relay is unnecessary. `dev:web` starts only Vite.

### npm Scripts

```json
{
  "test:e2e": "playwright test --config e2e/playwright.config.ts",
  "test:e2e:ui": "playwright test --config e2e/playwright.config.ts --ui"
}
```

The `--ui` flag launches Playwright's interactive test runner with time-travel debugging — useful for exploratory investigation when a test fails.

## Test Isolation

Playwright creates a **fresh BrowserContext per test** by default (via the `page` fixture). This means:
- **localStorage** is isolated between tests — no cleanup needed
- **WebSocket state** is isolated — each test gets its own connection
- **Network mocks** are per-page — no interference between parallel tests

For bookmark persistence tests specifically, save a bookmark then reload within the same test (same context = same localStorage).

## Timing & Flakiness Mitigation

Several app behaviours involve timing that tests must handle:

| Behaviour | Timing | Test Strategy |
|---|---|---|
| `flyTo` / `fitBounds` animations | 1-2 seconds real time | Assert status pill shows "Flying to..." (wiring test). Don't wait for animation completion. |
| URL hash debounce | 400ms after `moveend` | Use `page.waitForFunction(() => location.hash.includes('lat='))` — NOT `setTimeout` |
| Traffic UI injection | After async `buildMapStyle()` | Use `page.waitForSelector('[data-traffic-toggle="aircraft"]')` before interacting |
| Traffic activation | Only above zoom 5 | Navigate to `#z=8` in traffic tests to ensure traffic mocks are actually called |
| Overlay API fetches | After toggle enabled | API mocks return immediately; no timing concern |
| Status pill transitions | Sync with DOM updates | Use `await expect(locator).toHaveText('expected')` which auto-retries |

## What We Explicitly Skip

- **Visual regression / screenshot comparison**: WebGL rendering varies by GPU, driver, and OS. Not worth the flake rate. Unit tests already cover overlay geometry math.
- **Performance testing**: Not a correctness concern for this suite.
- **Mobile viewports**: The app doesn't have responsive breakpoints yet.
- **Multi-browser matrix**: Chromium only. `page.routeWebSocket()` is Chromium-specific.
- **Testing MapLibre internals**: We test our wiring and DOM state, not that MapLibre renders tiles correctly or that projections switch visually. If MapLibre works, our assertions about DOM text, classes, and attributes validate the integration.
- **Three.js / WebGL2 rendering**: The `Aircraft3dController` initializes in headless Chromium via SwiftShader. Console warnings from Three.js are expected and ignorable.
- **Flight route popups**: The "Show route" feature on aircraft popups is not covered. The route lookup, airport coordinate lookup, and great-circle rendering involve complex map interactions that are better verified through unit tests.
